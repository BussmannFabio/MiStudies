#### `analisar_pdf_professor.md`
**Prioridade:** P1 — Bloqueante para começar qualquer projeto

**Quando usar:** sempre que um professor envia material para ser transformado em aula.
Essa é a "Skill de Análise B2B" mencionada no relatorio-todos item 1.3.

**Entradas:**
- PDF ou texto do material do professor
- Tier contratado (Essencial / Pro / Full Studio)
- Respostas do briefing (se disponível)

**Saídas:**
- Mapa de estrutura lógica do conteúdo (tópicos, subtópicos, hierarquia)
- Avaliação de densidade: o que está completo vs. o que precisa ser complementado
- Lista de lacunas pedagógicas (o que falta para uma aula completa)
- Recomendação de estrutura para a aula (seções, ordem, destaques)
- Estimativa de complexidade de produção (fácil / médio / complexo)
- Flag de alertas: conteúdo desatualizado, ambíguo, ou que precisa de validação

**Comportamento esperado:**
```
1. Leia o material completo sem gerar ainda
2. Mapeie: quantos conceitos principais? Qual profundidade? Há fórmulas? Há exemplos?
3. Identifique o que o aluno PRECISA entender vs. o que é detalhe secundário
4. Proponha uma estrutura de aula em blocos (ex: Contexto → Conceito → Aplicação → Revisão)
5. Liste o que precisará ser pesquisado/complementado além do material original
6. Entregue o relatório de análise antes de produzir qualquer linha de HTML
```

**Formato de saída:** markdown estruturado com seções: Mapa, Lacunas, Estrutura Proposta, Alertas, Próximo Passo.

#### `produzir_aula.md`
**Prioridade:** P1

**Quando usar:** após análise aprovada e briefing completo. Produz o HTML da aula.
Uma skill única com o tier como parâmetro (não 3 skills separadas).

**Entradas:**
- Output de `analisar_pdf_professor.md`
- Tier: Essencial | Pro | Full Studio
- Briefing do professor (nome, área, matéria, nível do aluno, tom)
- Material complementar adicional (se houver)

**Saídas:** arquivo HTML completo, pronto para publicação em `pages/acervo/[slug].html`

**Escopo por tier:**

| Elemento | Essencial | Pro | Full Studio |
|---|---|---|---|
| Hero com título e metadados | ✅ | ✅ | ✅ |
| Seções de conteúdo | 3–5 | 5–8 | 8–12+ |
| Tabelas e comparativos | básico | intermediário | avançado |
| Fórmulas (KaTeX) | quando necessário | ✅ | ✅ |
| Exercícios de fixação | ❌ | 3–5 questões | 5–10 com gabarito |
| Interatividade JS | ❌ | básica (toggle/tabs) | rica (calculadoras, quizzes) |
| Citações acadêmicas | ❌ | referências inline | seção bibliográfica completa |
| Linha do tempo / diagrama | ❌ | ❌ | ✅ |
| Design editorial premium | padrão | padrão+ | customizado |

**Comportamento esperado:**
```
1. Carregue o contexto do tier selecionado
2. Use a estrutura proposta pela análise como esqueleto
3. Siga os tokens Tailwind e componentes do design system (ver Brandbook)
4. Nunca inventar fatos — se o material não cobre, diga que precisa de input adicional
5. Entregue o HTML completo com todos os metadados preenchidos
6. Ao final, liste: o que foi produzido, o que foi complementado com pesquisa, o que precisaria de revisão do professor
```

#### `revisar_aula.md`
**Prioridade:** P1

**Quando usar:** antes de entregar qualquer aula ao professor. QA obrigatório.

**Entradas:** arquivo HTML da aula produzida

**Saídas:** relatório de revisão com status por critério + lista de correções necessárias

**Checklist que a skill executa:**

```
CONTEÚDO
[ ] Todos os conceitos-chave do briefing estão presentes?
[ ] Nenhuma afirmação factualmente incorreta ou desatualizada?
[ ] Exemplos são concretos e fazem sentido para a área?
[ ] Nível de complexidade condiz com o tier e o público?

ESTRUTURA PEDAGÓGICA
[ ] A aula tem progressão lógica (do geral para o específico)?
[ ] Há introdução contextualizadora?
[ ] Há síntese/revisão ao final?
[ ] Os exercícios (Pro/Full Studio) cobrem os conceitos centrais?

DESIGN E HTML
[ ] Todos os tokens Tailwind corretos (primary, surface-container-*, etc.)?
[ ] Tipografia: Manrope nos títulos, Inter no corpo?
[ ] Nav com links corretos (../../index.html, ../acervo.html)?
[ ] Nenhum link quebrado ou placeholder "#"?
[ ] Responsivo em mobile (verificar breakpoints)?

METADADOS
[ ] <title> preenchido corretamente?
[ ] <meta description> tem 120–160 caracteres?
[ ] Slug no padrão kebab-case?
[ ] Tier badge e formato badge corretos?

QUALIDADE FINAL
[ ] Aula parece parte do sistema MiStudies (tom, visual, consistência)?
[ ] Nível de produção condiz com o tier cobrado?
```

#### `gerar_metadados_acervo.md`
**Prioridade:** P1

**Quando usar:** para gerar o objeto JS que vai no array `CONTEUDOS` do `acervo.html`.

**Entradas:** aula HTML finalizada (ou descrição da aula)

**Saídas:** objeto JS completo pronto para colar no `acervo.html`:

```javascript
{
  id: [próximo id disponível],
  slug: '[gerado automaticamente]',
  titulo: '[extraído do H1 da aula]',
  descricao: '[2-3 linhas, tom do Tom de Voz]',
  area: '[engprod | administracao | direito | ti]',
  materia: '[extraído do conteúdo]',
  tags: ['tag1', 'tag2', 'tag3', 'tag4'],
  prof: '[nome do professor]',
  prof_slug: '[slug ou null]',
  data: '[YYYY-MM-DD de hoje]',
  formato: '[Resumo | Aula | Slide | Masterclass]',
  tier: '[Essencial | Pro | Full Studio]',
  privacidade: '[publico | assinantes | privado]',
  arquivo: 'acervo/[slug].html'
}
```

**Comportamento esperado:**
```
1. Leia a aula HTML
2. Extraia título, conceitos-chave, nível, área
3. Gere slug em kebab-case (máx. 5 palavras, sem acentos)
4. Escreva descrição em 2–3 linhas no tom de voz do MiStudies
5. Sugira 4–6 tags relevantes para busca
6. Identifique área e matéria com base na taxonomia existente no acervo.html
7. Entregue o objeto JS formatado, pronto para colar
```

#### `briefing_professor.md`
**Prioridade:** P1

**Quando usar:** no início de cada projeto, antes de analisar o material.
Essa conversa define o escopo e evita retrabalho.

**Entradas:** contexto inicial do professor (nome, área, o que ele quer)

**Saídas:** documento de briefing estruturado com todas as informações necessárias para produção

**Perguntas que a skill faz (em conversa):**

```
SOBRE O PROFESSOR
- Qual é o seu nome completo e como prefere ser referenciado?
- Em qual instituição leciona? (nome + cidade)
- Qual é a disciplina ou tema desta aula?
- Para qual nível de aluno? (graduação, pós, técnico, profissional)

SOBRE O MATERIAL
- O que você está enviando? (PDF, slides, áudio, anotações)
- Qual é o objetivo principal desta aula? O aluno precisa SABER o quê ao final?
- Há algum tópico que você considera obrigatório, que não pode ficar de fora?
- Há algum erro comum que seus alunos cometem neste tema?

SOBRE O TOM E ESTILO
- Como você descreveria o tom da sua aula? (formal, dinâmico, técnico, acessível)
- Prefere exemplos mais acadêmicos ou mais práticos/empresariais?
- Há algum professor ou material de referência que você admira visualmente?

SOBRE ACESSO E PUBLICAÇÃO
- Esta aula será pública, apenas para assinantes ou privada?
- Quer seu nome e perfil vinculados à aula no acervo?
- Há alguma restrição de uso ou direito autoral no material enviado?

SOBRE O PROJETO
- Tier contratado: Essencial / Pro / Full Studio?
- Há prazo específico para entrega?
- Esta é a primeira aula de uma série ou uma peça avulsa?
```

**Formato de saída:** documento markdown com seções preenchidas, pronto para usar como contexto na produção.

#### `qualificar_lead.md`
**Prioridade:** P2

**Quando usar:** para avaliar rapidamente se um professor potencial é um bom fit antes de investir tempo em pitch.

**Entradas:** informações sobre o professor (cargo, área, instituição, o que ele tem, o que ele quer)

**Saídas:** score de qualificação + recomendação de abordagem

**Critérios avaliados:**

| Critério | Peso | Bom sinal | Sinal de alerta |
|---|---|---|---|
| Área de ensino | Alto | Eng. Produção, Adm, Direito, TI | Área muito nichada sem demanda |
| Volume de conteúdo | Alto | Disciplina regular com material próprio | "Não tenho nada escrito ainda" |
| Consciência do problema | Alto | "Meu material é bagunçado" | "Meus alunos amam meus slides" |
| Disposição para pagar | Alto | Já investe em ferramentas, quer resultado | "Quero ver primeiro de graça" |
| Nível de urgência | Médio | Semestre começando, prazo de entrega | Sem urgência declarada |
| Influência na instituição | Médio | Coordenador, professor sênior | Aluno de pós dando aula eventual |

**Score: 0–10** + uma das 3 recomendações: Abordar agora / Nutrir e revisitar / Não priorizar

#### `gerar_proposta.md`
**Prioridade:** P2

**Quando usar:** após briefing completo, para gerar a proposta comercial formal.

**Entradas:** documento de briefing preenchido + tier escolhido

**Saídas:** proposta em markdown com:
- Resumo do projeto (o que será feito)
- Escopo detalhado (o que está incluído e o que não está)
- Tabela de preços (Onboarding + aulas + Hub se aplicável)
- Prazo estimado por etapa
- Próximos passos (o que o professor precisa fazer para iniciar)
- Validade da proposta (7 dias)

### ÁREA 3 — Publicação e Distribuição

---

#### `checklist_publicacao.md`
**Prioridade:** P1

**Quando usar:** antes de fazer `git add` em qualquer nova aula. Processo obrigatório.

**Entradas:** arquivo HTML da aula revisada

**Saídas:** confirmação de que tudo está ok ou lista de itens pendentes

```
ARQUIVO HTML
[ ] Nome do arquivo no padrão: [slug].html (kebab-case, sem acentos)
[ ] Localizado em: pages/acervo/[slug].html
[ ] <title> preenchido: "[Título da Aula] — MiStudies"
[ ] <meta description> presente com 120–160 caracteres
[ ] Nav: logo → ../../index.html | Acervo → ../acervo.html

ACERVO.HTML
[ ] Objeto JS adicionado ao array CONTEUDOS[]
[ ] Todos os campos preenchidos (id, slug, titulo, descricao, area, materia, tags, prof, prof_slug, data, formato, tier, privacidade, arquivo)
[ ] Slug no objeto bate com o nome do arquivo
[ ] Campo `arquivo` no formato: 'acervo/[slug].html'

PERFIL DO PROFESSOR (se prof_slug não for null)
[ ] pages/professor/[prof_slug].html existe
[ ] Aula adicionada ao array AULAS[] do perfil do professor

DASHBOARD
[ ] Array AULAS[] no dashboard.html atualizado com a nova aula

GIT
[ ] git status limpo (sem arquivos esquecidos)
[ ] Commit mensagem: "Publicar aula: [título] ([tier])"
```

#### `gerar_post_social.md`
**Prioridade:** P3

**Quando usar:** para divulgar uma aula publicada nas redes sociais do MiStudies.

**Entradas:** link e metadados da aula publicada

**Saídas:** 3 versões de post prontas:

```
1. LINKEDIN (tom mais formal, foco em professores)
   - Abertura com dado ou insight do conteúdo
   - 3 bullet points do que o aluno aprende
   - CTA para o acervo
   - 3–5 hashtags relevantes

2. INSTAGRAM CAPTION (tom mais próximo, foco em alunos)
   - Abertura que captura atenção (pergunta ou afirmação forte)
   - Breve descrição do que a aula resolve
   - CTA: "Link na bio → Acervo MiStudies"
   - Emojis e hashtags

3. CAROUSEL ROTEIRO (5–7 slides)
   - Slide 1: título chamativo
   - Slides 2–5: um conceito-chave por slide, em linguagem simples
   - Último slide: CTA + logo MiStudies
```

#### `gerar_preview_card.md`
**Prioridade:** P2

**Quando usar:** para gerar a descrição de um card no acervo quando a aula já existe mas os metadados de marketing estão fracos ou genéricos.

**Entradas:** conteúdo da aula (HTML ou resumo)

**Saídas:**
- Título refinado (se o original for técnico demais)
- Descrição de 2–3 linhas no tom do MiStudies (clara, direta, com benefício explícito)
- 4–6 tags otimizadas para busca no acervo
- Frase de destaque para usar em preview (se a aula tiver um insight forte)

### ÁREA 4 — Aluno (B2C)

---

#### `gerar_exercicios.md`
**Prioridade:** P3

**Quando usar:** para complementar uma aula já produzida com exercícios de fixação (pode ser adicionado a uma aula Pro ou Full Studio).

**Entradas:** conteúdo da aula + nível desejado (básico / intermediário / avançado)

**Saídas:** bloco HTML com 5–10 exercícios, no formato:
- Múltipla escolha (4 alternativas, gabarito comentado)
- Cálculo com passo a passo
- Questão dissertativa com resposta-modelo
- Estudo de caso curto

**Comportamento:** integrado com o tom e taxonomia de Bloom — cada exercício deve testar um nível diferente (memorização → compreensão → aplicação → análise).

---

#### `gerar_trilha_estudo.md`
**Prioridade:** P3

**Quando usar:** para sugerir uma sequência de aulas do acervo para um aluno com objetivo específico.

**Entradas:** objetivo do aluno (ex: "entender Eng. de Produção do zero", "revisar para prova de PCP")

**Saídas:**
- Trilha ordenada de conteúdos do acervo com justificativa de sequência
- Tempo estimado total
- Pré-requisitos (se houver)
- Dicas de estudo por etapa
